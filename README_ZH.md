# DBAIOps
当前仓库继续沿用 **DBAIOps** 的代码历史，但工作区内容已经从原先的服务端运维平台部署包，调整为基于浏览器扩展的数据库知识问答与分析工作区。当前工作区已经切换为基于浏览器扩展的 DBAIOps 形态，并继续沿用现有仓库历史。

当前 DBAIOps 主要聚焦以下能力：

- **数据库知识问答：** 面向主流数据库与国产数据库提供知识库检索与问答。
- **AWR 报告分析：** 解析 Oracle AWR 报告并生成结构化分析结果。
- **巡检分析：** 支持巡检任务查看、历史记录回溯与问题汇总。
- **SQL 优化辅助：** 支持 SQL 优化分析与报告脱敏相关流程。
- **模型与服务接入：** 支持配置知识库服务地址、API Key 与模型服务商。

## 1. 仓库概览

### 1.1 核心能力

| 模块 | 说明 |
|------|------|
| 知识问答 | 支持 SQL 语法、故障排查、性能调优、最佳实践等知识检索 |

## 2. 本地加载与基础配置

### 2.1 加载未打包扩展

1. 打开浏览器扩展管理页面。
2. 启用开发者模式。
3. 点击 **加载已解压的扩展程序**。
4. 选择以下目录：
```bash
/data/wei/program/DBAIOps/DBAIOps_v1.1.1
```
5. 确认扩展图标、弹窗页面与设置页面能够正常打开。

### 2.2 注册与知识库服务配置

#### 步骤 1：用户注册
打开设置页面后填写用户信息。扩展默认会以 `config/registration.json` 作为注册服务模板，并在浏览器本地存储中保存个性化配置。

#### 步骤 2：知识库服务配置
进入知识库服务配置页面，填写 API Key 与服务地址。默认模板来源如下：
```bash
DBAIOps_v1.1.1/config/knowledge_service.json
```

#### 步骤 3：配置校验
保存后建议确认以下事项：
- 能够正常选择目标知识库；
- 配置的服务地址可访问；
- 弹窗页面能够正常提交问题并返回结果。

### 2.3 模型服务商配置

当前 DBAIOps 通过浏览器端页面完成模型服务商配置。

常见流程如下：
1. 打开 **设置** -> **模型与服务商**。
2. 新增 `ollama`、`deepseek` 或其他兼容 OpenAI 接口的服务商。
3. 填写 API Base URL 与 API Key（如需要）。
4. 使用页面内测试功能验证连通性，并保存选中的模型。

**本地 Ollama 示例：**
```bash
http://localhost:11434/v1
```

## 3. 分析能力说明

### 3.1 AWR 报告分析

DBAIOps 当前保留了扩展内的 Oracle AWR 报告分析流程。

**支持的数据来源：**
- Oracle 单实例 AWR HTML 报告
- 通过 `awrrpt.sql` 或 `awrrpti.sql` 生成的 RAC AWR 报告

**当前说明：**
- `awrddrpi.sql` 生成的 AWR 对比报告不是当前主要目标场景。
- `awrgrpt.sql` 生成的全局报告建议在正式使用前先完成验证。

**典型流程：**
1. 打开 AWR 分析面板。
2. 填写问题概述和接收邮箱（如需要）。
3. 上传 AWR 报告文件。
4. 选择分析语言。
5. 提交任务并在历史记录中查看结果。

### 3.2 巡检分析

当前扩展同样保留了数据库巡检分析流程，适合在运维回顾、周期性巡检和历史问题跟踪场景中使用。

常见用途包括：
- 查看周期性巡检任务；
- 检查生成的摘要与异常项；
- 在模型或服务配置变更后重新分析历史任务。

### 3.3 SQL 优化与脱敏

DBAIOps 继续保留了扩展中的 SQL 优化与报告脱敏相关能力。

主要适用于：
- 提交 SQL 语句进行优化分析；
- 在上传报告前降低敏感 SQL 文本暴露风险；
- 为 DBA 与数据库工程师保留浏览器端分析流程。
