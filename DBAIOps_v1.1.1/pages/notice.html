DBAIOps v1.1.1 版本更新

全新功能上线，体验全面升级
Version 1.1.1

🎊 版本亮点

本次更新带来了多项重要功能优化和体验提升，涵盖 AWR 分析、巡检诊断、SQL 优化等核心模块，以及整体性能和用户体验的全面优化。

AWR 分析模块新功能

📊 脱敏脚本下载：新增 AWR/KWR 脱敏脚本下载功能，支持 Oracle AWR 和 KingBase KWR 报告的脱敏处理，有效保护敏感信息，提升数据安全性。
🔗 便捷下载链接：提供 Gitee 和 GitHub 双平台下载地址，支持一键跳转，下载更便捷。
📋 任务状态优化：优化历史记录筛选功能，任务状态下拉项顺序调整为：全部、执行中、成功、失败，查找更高效。
💡 智能提示：根据选择的数据库类型（Oracle/KingBase）自动显示对应的脱敏脚本类型和说明文案。

巡检诊断模块新功能

📥 脚本采集器优化：重构脚本采集器下载方式，提供 Gitee 和 GitHub 双平台下载链接，支持根据选择的数据库动态显示对应的脚本采集器下载地址。
📝 详细说明文档：为脚本采集器添加详细的功能说明，帮助用户更好地理解和使用采集工具。
🎨 界面优化：优化下载链接的展示方式，采用上下排列布局，视觉效果更清晰。
⚡ 性能提升：优化初始化流程，减少不必要的延迟，提升页面加载速度。

SQL 优化模块新功能

📥 脚本采集器优化：与巡检诊断模块同步优化，提供统一的脚本采集器下载体验，支持 Gitee 和 GitHub 双平台。
📚 功能说明：添加 SQL 优化采集脚本的详细说明，帮助用户了解脚本的用途和使用场景。
🔍 历史记录优化：优化任务状态筛选功能，提升查询效率。

其他重要优化

  • 性能优化：全面优化插件启动速度，减少初始化延迟，点击插件图标后响应更快。
  • 错误处理：优化网络请求错误处理机制，添加超时控制，避免长时间等待阻塞操作。
  • 用户体验：优化界面交互，改进错误提示方式，减少不必要的控制台错误信息。
  • 代码优化：重构部分核心代码，提升代码质量和可维护性。
  • 稳定性提升：修复多个已知问题，提升插件整体稳定性和可靠性。
